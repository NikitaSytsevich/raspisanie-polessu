# Расписание

Веб-приложение для отображения расписания спортивных объектов Полесского государственного университета (г. Пинск) с приоритетом под iOS-устройства.

Приложение собирает данные с официальных страниц университета, нормализует расписание по датам и показывает его в компактном таймлайне. Дополнительно есть раздел **«Мой график»** для личных смен инструктора с верификацией по фактическим сеансам сайта.

## Что умеет приложение

- Подтягивает расписание 4 объектов:
  - Ледовая арена
  - Большой бассейн
  - Малый бассейн
  - Гребная база
- Показывает расписание «сегодня → дальше по датам», с удобной навигацией по дням.
- Подсвечивает текущие/будущие/прошедшие сеансы.
- Рассчитывает и отображает перерывы между сеансами.
- Учитывает специфику гребной базы: суббота и воскресенье отмечаются как выходные дни.
- Поддерживает светлую и тёмную темы.
- Раздел **«Мой график»**:
  - добавление/удаление смен;
  - метрики по месяцу;
  - «ближайшая смена» только по реально существующим на сайте сеансам;
  - для статуса «Частично» показывает только реальные сеансы из заданного интервала и перерывы между ними;
  - импорт/экспорт истории смен в JSON.

## Архитектура

Проект без сложного backend-стека, рассчитан на serverless-подход:

- **Frontend (статический):**
  - `index.html`
  - `styles.css`
  - `app.js`
- **Serverless API-парсер:**
  - `api/schedule.js` (Vercel Function)
- **Локальный dev-сервер:**
  - `dev-server.js` (раздаёт статику + проксирует `/api/schedule`)

### Кэширование

- На стороне API: in-memory cache на 5 минут.
- На стороне клиента: `localStorage` (последний успешный payload + пользовательские настройки/смены).

## API

### `GET /api/schedule`

Возвращает нормализованное расписание по всем объектам.

Параметры:

- `refresh=1` — принудительно игнорировать in-memory cache и запросить источники заново.

Пример:

```bash
curl "http://localhost:3000/api/schedule?refresh=1"
```

Ключевые поля ответа:

- `generatedAt` — время генерации сводного ответа.
- `timezone` — `Europe/Minsk`.
- `facilities[]` — список объектов с массивом `days[]` и `sessions[]`.
- `meta.cached` — признак, что ответ пришёл из server cache.

## Локальный запуск

Требования:

- Node.js 18+

Команды:

```bash
npm install
npm run dev
```

После запуска сервер печатает ссылки для:

- `http://localhost:3000`
- локальной сети (LAN) — удобно для проверки на iPhone в одной Wi‑Fi сети.

## Ограничения и особенности

- Парсинг зависит от структуры страниц источника; при сильных изменениях HTML может понадобиться обновление правил.
- Для гребной базы источник публикует интервалы без явной привязки к датам, поэтому используется шаблон на ближайшие дни с выходными в субботу/воскресенье.
- История «Моего графика» хранится локально в браузере пользователя (без общей серверной БД).
