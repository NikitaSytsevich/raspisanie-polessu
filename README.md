# Расписание ПолесГУ

Веб-приложение для личного графика инструктора с синхронизацией расписания спортивных объектов Полесского государственного университета.

Текущая версия сфокусирована на разделе **«Мой график»**:
- основной экран по выбранным суткам;
- отдельная страница редактирования смен;
- проверка изменений на официальном сайте;
- настройки темы/автообновления и импорт/экспорт истории.

## Основные возможности

### 1) Мой график (главный экран)
- По умолчанию приложение открывается на странице `Мой график`.
- Основной акцент: график на выбранные сутки.
- Режимы просмотра:
  - `DAY` — только выбранные сутки;
  - `WEEK` — выбранные сутки + 7 дней вперёд;
  - `FULL` — полный график от выбранной даты.
- Если история пустая, показывается пустое состояние с кнопками:
  - `Импорт истории`;
  - `Добавить вручную`.
- Внизу — интерактивная краткая карточка `Изменения на сайте`.

### 2) Редактор смен (отдельная страница)
- Добавление, изменение и удаление смен.
- Поля смены:
  - дата;
  - объект;
  - время начала/окончания;
  - комментарий;
  - `С кем работаю` (выбор инструкторов).
- Кнопка `Удалить историю` с подтверждением.

### 3) Изменения на сайте
- Любая успешная синхронизация считается проверкой и обновляет локальный мониторинг.
- Первый успешный запрос создаёт `Базовый снимок`, дальше идут сравнения с предыдущей проверкой на этом устройстве.
- Верхний блок показывает, затрагивают ли изменения ваши смены, сколько объектов/дат изменилось и есть ли проблемы источников.
- Ниже отображаются:
  - влияние на мои смены;
  - все изменения текущей проверки;
  - локальный журнал событий без скрытия просмотренных записей.
- Кнопка `Просмотрено` помечает запись журнала, но не удаляет её из ленты.
- Для шаблонных источников (например, гребная база) сравнение ведётся по шаблону, а не по сдвигающемуся окну дат.
- Клик по карточке изменения открывает официальный источник.

### 4) Настройки
- Тема: `Светлая`, `Тёмная`, `Системная` (по умолчанию — системная).
- Автообновление расписания.
- Импорт/экспорт JSON истории смен.
- Экспорт включает также историю изменений сайта.

## Источники данных

Парсер читает официальные страницы ПолесГУ:
- Ледовая арена
- Большой бассейн
- Малый бассейн
- Гребная база

Особенности парсера:
- базовые URL + fallback-поиск через sitemap (`https://www.polessu.by/sitemap.xml`);
- при недоступности источника используется безопасная fallback-логика:
  - если есть предыдущие данные, возвращается `stale_cache`;
  - если данных нет, возвращается `error` без ложных изменений;
- для гребной базы используется шаблон ближайших дней (с отметкой выходных).

## API

### `GET /api/schedule`
Возвращает нормализованное расписание по объектам.

Параметры:
- `refresh=1` — принудительно обойти in-memory cache.

Пример:
```bash
curl "http://localhost:3000/api/schedule?refresh=1"
```

Ключевые поля ответа:
- `schemaVersion`
- `generatedAt`
- `sourceCheckedAt`
- `snapshotHash`
- `timezone` (`Europe/Minsk`)
- `facilities[]`
- `facilities[].comparisonMode`
- `facilities[].dataQuality`
- `facilities[].facilityHash`
- `facilities[].changeTrackingHash`
- `meta.cached`
- `meta.sourceIssueCount`
- `meta.sourceIssues[]`

## Локальный запуск

Требования:
- Node.js 18+

Установка и старт:
```bash
npm install
npm run dev
```

Сервер поднимает:
- `http://localhost:3000`
- LAN-адреса текущей сети (показываются в консоли)

Запуск с доступом из локальной сети:
```bash
HOST=0.0.0.0 PORT=3000 npm run dev
```

Если порт занят:
```bash
lsof -nP -iTCP:3000 -sTCP:LISTEN
kill -9 <PID>
```

## Формат импорта/экспорта

Экспорт создаёт JSON формата:
```json
{
  "version": 4,
  "app": "Расписание",
  "exportedAt": "ISO_DATE",
  "timezone": "Europe/Minsk",
  "shifts": [],
  "siteChanges": {
    "lastCheckedAt": "ISO_DATE_OR_EMPTY",
    "acknowledgedSignature": "",
    "history": []
  }
}
```

Поддерживается импорт:
- массива смен;
- объекта с полем `shifts`;
- опционального блока `siteChanges`.

## Хранение данных

Клиентские данные хранятся в `localStorage`:
- настройки;
- кэш последнего расписания;
- история смен;
- история изменений сайта и отметка просмотра записей журнала.

## Ограничения

- Парсинг зависит от разметки внешних страниц ПолесГУ.
- История смен/изменений хранится локально в браузере (без общей серверной БД).
- При резком изменении структуры источника может потребоваться обновление правил парсинга.
